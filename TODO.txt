[ ] Backfill flat-stored descriptions (optional but worthwhile)
    Jobs ingested before this fix have no paragraph structure in the DB.
    The extractor handles them via sentence-level fallback, but quality
    is lower. To fix them properly:

      uv run mcf backfill-descriptions   # re-fetch raw HTML from MCF API
      uv run mcf re-embed --db data/mcf.duckdb

    Run backfill-descriptions with --limit N for batched runs to avoid
    rate limiting. The re-crawl will apply the improved clean_description().

PHASE 2 — OPTIONAL LLM CLEANING (future)
------------------------------------------------------------

[ ] Implement ClaudeJobCleaner (LLM extractive cleaner)
    The interface is already in job_description_extractor.py.
    To wire it up:

    1. pip install anthropic (add to pyproject.toml)
    2. Create src/mcf/lib/embeddings/llm_job_cleaner.py
       Implement LLMJobCleaner protocol:
         - should_clean(): return True if word_count > threshold
           (default 800, set via JOB_EXTRACTOR_LLM_THRESHOLD)
         - clean(): call Claude Haiku with an extractive prompt:
           "Extract only requirements, qualifications, responsibilities,
            and tools from the following job description. Preserve exact
            wording. Do not summarize or add content not in the original."
         - Cache the cleaned result to avoid repeated API calls
    3. Register at app startup (e.g. in server.py lifespan):
         from mcf.lib.embeddings.llm_job_cleaner import ClaudeJobCleaner
         register_llm_cleaner(ClaudeJobCleaner())
    4. Set JOB_EXTRACTOR_LLM_ENABLED=1 in your .env
    5. Run re-embed to process existing jobs

    Cost estimate: Claude Haiku is ~$0.25/M input tokens.
    At ~500 tokens per job description and 10k jobs = ~$1.25 total.
    Only runs for jobs above the word threshold anyway.
